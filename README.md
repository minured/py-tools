# py-tools

Small Python utility scripts.

## Overview

This repository currently contains a scheduled automation script for the China Unicom Friday welfare mall flow. The script:

- Fetches available goods from the activity API.
- Places repeated order requests for configured goods.
- Sends email notifications with execution results.
- Uses scheduled jobs to run reminders and order placement at configured times.

## Requirements

- Python 3.8+
- Network access to the target APIs
- A valid login cookie for the activity endpoint
- SMTP credentials for email notification

Python dependencies:

```bash
pip install requests schedule
```

## Configuration

Before running the script, review and configure the following values in `src/meituan30.py`:

- `headers["Cookie"]`: login cookie for the China Unicom client API.
- `mail_host`: SMTP server host.
- `mail_user`: SMTP login account.
- `mail_pass`: SMTP authorization code or password.
- `sender`: sender email address.
- `receivers`: recipient email address.
- `goodsID`: target goods IDs.
- Phone number passed to `order(...)`.
- Scheduled execution times in the `schedule.every()` calls.

> Important: do not commit real cookies, tokens, passwords, phone numbers, or other sensitive data. Prefer storing them in environment variables or an ignored local config file.

## Usage

Run the script from the repository root:

```bash
python src/meituan30.py
```

The script starts a long-running scheduler loop. Keep the process alive until the scheduled jobs finish.

## Notes

- The script depends on session state. If the cookie expires, refresh it before the scheduled run.
- Email sending requires a working SMTP account and authorization code.
- API parameters and goods IDs may become invalid if the upstream activity changes.
- `result.html` may be generated by the helper function and should be treated as runtime output.
